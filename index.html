Git e GitHub

A principal diferença entre Git e o GitHub é que o primeiro é utilizado para controle de versão, oferecendo maior segurança para aqueles que trabalham no mesmo código sem que haja grandes confusões. O GitHUb permite que desenvolvedores troquem conteúdos e se conectem, funcionando de maneira semelhante a uma rede social.

Tutorial

Existem algns comandos que são fundamentais para o trabalho com o Git. São eles:

Comandos mais usados

  • ls: lista tudo o que tem no computador;
  • pwd: permite identificar o caminho seguido até o momento;
  • cd "nome": serve para abrir pastas, para se locomover;
  • cd ~: permite voltar para a raiz;
  • cd ..: volta uma pasta acima;
  • touch nome-do-arquivo.extensão: permite voltar para a raiz;
  • mkdir nome-do-diretorio: cria diretório;
  • mv: mover (renomear);
  • man: para mostrar o manual;
  • whoami: identifica o usuário que está mexendo no sistema;

    • Usando o Git no terminal

  • git config -- globaluser.name "NOME": permite que o Git identifique quem está usando o sistema;
  • git config -- globaluser.email "EMAIL":permite o reconhecimento do email de quem está usando o GIT;

    • Como criar um repositório

    Para criar um repositório é necessário que se crie uma conta no GitHub. Finalizado este processo um novo repositório pode ser criado clicando no sinal de + ao lado de sua foto de perfil. Uma nova tela será aberta onde constarão os dados desse novo repositório, como quem criou, nome do repositório, descrição.
